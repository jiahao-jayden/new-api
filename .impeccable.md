## Design Context

### Users

The user-facing console serves ordinary users, including people creating an API key for the first time. They want to choose a supported model family, create a key, and copy it into their application with minimal setup knowledge. The onboarding aha moment is receiving a usable key with a clear name and understanding what it is for.

### Brand Personality

Calm, precise, and approachable. The interface should feel trustworthy and quietly polished, with concise language and enough guidance for beginners without slowing down experienced users.

### Aesthetic Direction

Use refined Apple-inspired product UI principles: disciplined typographic hierarchy, generous but purposeful whitespace, crisp alignment, restrained color, and responsive controls. Preserve the existing theme tokens and light/dark mode behavior. Avoid oversized type, decorative card grids, excessive shadows, dense technical labels, and visual effects that compete with the task.

### Design Principles

1. Reveal one decision at a time and keep the shortest path to a usable result.
2. Use smart, editable defaults so beginners can continue without configuration.
3. Establish hierarchy through type, weight, contrast, and spacing rather than decoration.
4. Keep primary actions obvious, secondary actions quiet, and every touch target at least 44px.
5. Preserve accessibility, keyboard operation, theme compatibility, and responsive behavior.
